122 // Use the same accelerator id for all hardware blocks. This means we will
123 // simulate only ONE datapath instead of multiple, which means that the two
124 // blocks can share the scratchpads (without any infrastructure
125 // changes). The key is that we still trace the functions at the _hw level,
126 // so that Aladdin will exit after simulating each block, and we can return
127 // control to the CPU at the right places. In contrast, if we used two
128 // different ids, we would have two different datapaths that could not share
129 // data directly.
